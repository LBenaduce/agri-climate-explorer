# Agri Climate Explorer

Full-stack web application for searching weather conditions, showing agricultural insights, and saving favorite locations.

## Stack
- Frontend: React + Vite
- Backend: Node.js + Express + MongoDB
- Authentication: JWT
- Styling: CSS with reusable components

## Main features
- Search weather by city
- User registration and login
- Protected routes for authenticated users
- Save and delete favorite locations
- Agricultural insights based on temperature, humidity, rainfall, and wind
- Responsive interface with language selection

## Project structure

```bash
agri-climate-explorer/
  frontend/
  backend/
```

## Running the project locally

### 1. Backend
```bash
cd backend
npm install
cp .env.example .env
npm run dev
```

### 2. Frontend
Open a second terminal:

```bash
cd frontend
npm install
cp .env.example .env
npm run dev
```

## Environment variables

### Backend (`backend/.env`)
```env
PORT=3001
MONGO_URI=mongodb://127.0.0.1:27017/agriclimate
JWT_SECRET=change-me-in-production
```

### Frontend (`frontend/.env`)
```env
VITE_API_URL=http://localhost:3001
```

## API routes
- `POST /signup` — create user
- `POST /signin` — login and receive JWT
- `GET /users/me` — get current user
- `GET /weather?city=NAME` — get weather data for a city
- `GET /locations` — list saved locations
- `POST /locations` — save a location
- `DELETE /locations/:id` — delete a saved location

## Notes
- The current weather route returns demo climate data generated by the backend for development purposes.
- MongoDB must be running locally before starting the backend.
- The frontend expects the backend to run on port `3001` by default.
